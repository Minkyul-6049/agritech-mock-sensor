#Smart Farm Edge Sensor & Control System

## Project overview
This project is structured by simulating greenhouse environment. 
This system collecting sensor data which generated by edge device and visualizing data pipeline is configured.
I configured miniature of smart farm not only collecting data, but also based on live-accumulated data.

## Architecture Flow
1. Edge node(Farm-node) : I have made virtual daemon which is collecting every 2 seconds, humidity, temperature and soil moisture
2. Control Logic : Upon detecting anomalies that require immediate intervention (e.g., low soil moisture), the edge logic automatically triggers virtual actuators like sprinklers or ventilation.
3. Time Series(Monitor-node) : Under K3s environment, Live data transferring by influxDB.
4. Visualization : Live-monitoring via grafana able to watch overall greenhouse status.

## Used Tech Stack 
Edge App: Golang
Infrastructure : K3s (Lightweight Kubernetes)
Database : InfluxDB v2.
Monitoring : Grafana
OS/ENV : Linux, systemd (Daemon)

## Key Features
Influx DB live data transfer including temperature, humidity & soil moisture
Sprinkler operation in case of shortage on soil moisture
ventilation operation in case of high temperature detected
Auto-restoration & Background working utilized by linux systemd

# 🚜 Smart Farm Edge-to-Cloud Data Pipeline

This repository demonstrates a complete edge-to-cloud IoT data pipeline designed for Agritech environments. It features a lightweight Golang-based mock sensor, containerized and deployed on a Kubernetes (K3s) cluster to simulate real-time agricultural data collection and monitoring.
## 🏗️ Architecture Overview

The infrastructure is built with a focus on cloud-native principles, separating the edge computing node from the monitoring plane. It implements an Event-Driven, Closed-Loop automation system.
